Address Sequencing in Microprogrammed Control Units

Interactive Learning Experience

🎯 Introduction

In a digital computer, the control unit's primary function is to initiate a sequence of operations. When a control unit is microprogrammed, it uses microcode – a lower-level control program – to implement the necessary control signals for executing instructions.

Address sequencing refers to the capabilities required within a microprogrammed control unit to determine the order in which microinstructions are fetched and executed.

🔧 Key Capabilities Required

1. Incrementing CAR

Advancing to the next microinstruction in sequential program flow

2. Conditional/Unconditional Branching

Changing sequence based on conditions or unconditionally

3. Instruction Mapping

Translating opcode to control memory address

4. Subroutine Support

Call and return facility for microprogram subroutines

🏗️ System Components

Control Memory (ROM) 📁
Function: Stores microprograms - groups of microinstructions that implement computer instructions. Each word specifies a control word/microinstruction.
Control Address Register (CAR) 📍
Function: Holds the address of the next microinstruction to be fetched. Central to determining microprogram execution flow.
Next-Address Generator 🔄
Function: Determines address sequence for control memory. Handles CAR incrementing, external address loading, and processes mapping/branch logic.
Subroutine Register (SBR) 💾
Function: Stores return address during subroutine calls, enabling proper return to calling routine after subroutine completion.
Control Data Register (CDR) 🚀
Function: Pipeline register holding current microinstruction, allowing simultaneous execution and next-address generation for performance optimization.

🔄 Detailed Operations

CAR Incrementing

The simplest form of address sequencing where CAR is incremented by one to fetch the next sequential microinstruction.

Current CAR = n
Increment Operation
New CAR = n + 1
Fetch Next Instruction

Conditional & Unconditional Branching

Allows microprogram flow to jump based on status bits or unconditionally.

Branch Operations:

  • JMP: Jump to address if condition met, else increment CAR
  • Unconditional: Always branch (condition = 1)

Status bits tested: I bit - DR(15), S bit - AC(15), Z bit - AC=0, etc.

Instruction Mapping

Transforms instruction opcode bits into control memory addresses.

4-bit Opcode
Mapping Logic
7-bit Address
Routine Start

Example: DR(11-14) → CAR(2-5), CAR(0,1,6) = 0

Subroutine Call & Return

Supports reusable microprogram routines for common operations.

CALL Operation:

  • Save CAR + 1 to SBR (return address)
  • Load target address to CAR
  • Begin subroutine execution

RET Operation:

  • Load SBR contents to CAR
  • Resume calling routine

📝 Microinstruction Format

Field Breakdown:

CD Field (2 bits)
Condition for branching
BR Field (2 bits)
Branch operation type
AD Field (7 bits)
Target address

BR Field Values:

  • 00 - JMP: Jump if condition met
  • 01 - CALL: Subroutine call if condition met
  • 10 - RET: Return from subroutine
  • 11 - MAP: Map instruction bits to address
