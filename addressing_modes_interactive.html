What are Addressing Modes?

Addressing modes define the rules for interpreting or modifying the address field of an instruction before the operand is actually referenced. They provide programming versatility and help reduce the number of bits needed in the addressing field.

🎯

Programming Versatility

Provide pointers to memory, counters for loops, and indexing capabilities for data relocation.

⚡

Bit Reduction

Help reduce the total number of bits needed in the addressing field of instructions.

🔧

Flexible Access

Enable various methods for selecting operands during program execution.

Effective Address Formula
effective_address = address_part_of_instruction + content_of_CPU_register

The effective address is the actual memory location computed based on the addressing mode used.

Summary & Comparison

Quick Reference Table
Mode Syntax Effective Address Use Case
Immediate LD #100 Operand = 100 Constants
Register LD R1 EA = R1 Fast access
Register Indirect LD (R1) EA = M[R1] Pointers
Auto Increment LD (R1)+ EA = M[R1], R1++ Arrays/Lists
Direct LD 500 EA = 500 Global variables
Relative LD $50 EA = PC + 50 Branching
Indexed LD 500(X) EA = 500 + XR Arrays
📊

Performance Impact

Register modes are fastest, memory modes require additional cycles, indirect modes need multiple memory accesses.

🎯

Flexibility vs Speed

More flexible addressing modes often require more execution time but provide greater programming convenience.

🔍

Memory Efficiency

Some modes allow shorter instruction formats by reducing the address field size requirements.
