What is an Assembler?

An assembler is a crucial program that bridges the gap between human-readable assembly language and machine-executable binary code. It transforms symbolic representations into precise binary instructions that computer hardware can understand and execute.

The Two-Pass Assembly Process

The assembler uses a sophisticated two-pass approach to ensure accurate translation of symbolic code:

1

First Pass: Symbol Resolution

  • 🎯 Initialize Location Counter (LC) to 0
  • 🔍 Scan for Labels in each line
  • 📊 Build Symbol Table with addresses
  • ➕ Increment LC for each instruction
  • 🏁 Stop at END pseudoinstruction
2

Second Pass: Code Generation

  • 🔄 Reset LC to 0
  • 📋 Use Lookup Tables for translation
  • 🔢 Convert Instructions to binary
  • 🎯 Resolve Addresses from symbol table
  • 💾 Generate Object Code

Lookup Tables Used by Assembler

The assembler uses four essential tables during the second pass for accurate code translation:

📋 Pseudoinstruction Table

ORG, END, DEC, HEX

🎯 MRI Table

AND(0), ADD(1), LDA(2), STA(3)
BUN(4), BSA(5), ISZ(6)

⚙️ Non-MRI Table

CLA, CLE, CMA, CME, CIR, CIL
INC, SPA, SNA, SZA, SZE, HLT

🏷️ Address Symbol Table

User-defined labels
Generated in First Pass
