What is Assembly Language?

Assembly language is a programming language that uses symbolic representations instead of binary machine code. It provides a human-readable way to write programs that are then translated into binary code by an assembler. Each line of code follows a structured format with three distinct fields.

Assembly Language Structure

Each line in assembly language is organized into three fields:

1. Label Field
  • Purpose: Contains symbolic addresses
  • Format: 1-3 alphanumeric characters
  • Rule: First character must be a letter
  • Example: MIN, SUB, DIF
2. Instruction Field
  • Memory Reference: LDA, ADD, STA
  • Register Reference: CMA, INC, HLT
  • Pseudoinstructions: ORG, END, DEC, HEX
3. Comment Field
  • Purpose: Documentation and explanation
  • Format: Preceded by / symbol
  • Ignored: During binary translation
  • Example: / Load subtrahend to AC

Translation Process

The assembler performs a two-pass translation process:

1
First Pass: The assembler scans the program to identify all symbolic addresses and assign them memory locations. For example, MIN gets location 106, SUB gets 107, and DIF gets 108.
2
Second Pass: The assembler translates each instruction into binary/hexadecimal machine code using the address assignments from the first pass. LDA SUB becomes 2107 (hex).
