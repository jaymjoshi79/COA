Computer Design Overview

The design of a basic computer involves translating operational specifications into actual logic circuits. This process transforms register transfer statements into hardware implementations.

🎯 Key Design Objectives

Transform register transfer statements into functional hardware circuits that can execute computer instructions efficiently.

Control Unit Design

The control unit generates all necessary control signals for computer operation through decoders and logic gates.

🎛️ Control Functions

The control unit manages:

  • Register control inputs (LD, INR, CLR)
  • Memory read/write operations
  • Flip-flop state changes
  • Common bus register selection
  • Arithmetic and logic operations

📊 Address Register (AR) Control Example

Clear Operation:

CLR(AR) = RT0

Load Operation:

LD(AR) = R'T0 + R'T2 + D'7IT3

Increment Operation:

INR(AR) = D4T4

Control Signal Flow

🚌 Bus Control Logic

The 16-bit common bus is controlled by select inputs S2, S1, S0 of a multiplexer, determining which register's output is placed onto the bus. The control signals are derived from the register transfer statements.

Register Selection (S2, S1, S0)

  • 001: Output of AR
  • 010: Output of PC
  • 011: Output of DR
  • 100: Output of AC
  • 101: Output of IR
  • 110: Output of TR
  • 111: Memory M[AR]

Bus Selection Inputs (S2, S1, S0)

  • S0 = x1 + x3 + x5 + x7
  • S1 = x2 + x3 + x6 + x7
  • S2 = x4 + x5 + x6 + x7

Each binary value listed in the table corresponds to a Boolean variable xi, which activates the specific gate required to select the appropriate register or memory for the bus. When x1 = 1 to S₂S₁S₀ = 001 selects the AR register.

📊 Boolean Derivation Example

To determine the logic for each encoder input, it is necessary to find the control functions that place the corresponding register on the bus. For example, for x1:

D4T4: PC ← AR
D5T5: PC ← AR
x1 = D4T4 + D5T5

For x7, which represents memory selection:

x7 = R'T1 + D7'IT3 + (D0 + D1 + D2 + D6)T4
