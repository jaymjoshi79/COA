The Architecture of Control Memory

Exploring the role of ROM and RAM in storing the Binary Microprogram and defining the CPU's instruction set.

1. The Binary Microprogram

The control unit's operation is governed by a sequence of microinstructions known as the microprogram. While the symbolic form is human-readable, the Control Memory (CM) stores the Binary Microprogram—a translation of symbols into 1s and 0s.

2. ROM for Control Memory

A Read-Only Memory (ROM) is the most common choice for Control Memory due to its speed and cost-effectiveness.

  • Fabrication: The microprogram (the list of 1s and 0s) acts as the truth table for fabricating the ROM mask during hardware production.
  • Permanence: The microprogram is fixed. The internal links are fused, making the ROM content non-volatile and immutable.
  • Modification: Changing the instruction set requires generating a new microprogram, removing the old ROM package, and physically replacing it with the new one.
  • Security: Prevents the occasional user from inadvertently changing the fundamental architecture of the system.

3. RAM (Writable Control Memory)

When a RAM is used for the Control Memory, it is termed Writable Control Memory (WCM), offering significant flexibility.

  • Advantage: The microprogram can be altered simply by writing a new pattern of 1s and 0s. No hardware procedures are needed.
  • Flexibility: Allows the computer's instruction set to be changed dynamically by choosing which microprogram to load into the WCM.
  • Usage: Often employed in specialized or research architectures where frequent modification of the instruction set is required.
