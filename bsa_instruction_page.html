BSA: Branch and Save Return Address

Understanding Memory Reference Instructions

📋 Overview

The BSA (Branch and Save Return Address) instruction is a fundamental operation in computer architecture used for subroutine calling. It enables branching to a subroutine while preserving the return address for later continuation.

M[AR] ← PC, PC ← AR + 1

This instruction, expressed in register transfer notation, demonstrates how the return address is saved before jumping to the subroutine.

🎯 Instruction Tabs

Execution Process

1
The current value of the Program Counter (PC) is stored at the memory address specified by the Address Register (AR)
2
The effective address plus one (AR + 1) is transferred to PC
3
Program execution continues at the new PC address (subroutine entry point)
4
The return address is stored at the first location of the subroutine
⚠️ Important Note:

BSA cannot be executed in one clock cycle due to bus system limitations. It requires a sequence of two microoperations with appropriate timing signals.

💡 Key Insight:

The return address (21) is stored at the subroutine entry point (135), and the PC is set to 136 (AR + 1) to begin execution of the subroutine body.

Timing Sequence

The BSA instruction requires two microoperations:

D₅T₄: M[AR] ← PC, AR ← AR + 1
D₅T₅: PC ← AR, SC ← 0
⏱️ Timing Breakdown:
  • T₄: Initiates memory write, places PC content onto bus, enables INR input of AR
  • T₅: Completes memory write, transfers AR to PC by next clock transition

🔄 Subroutine Return:

The BUN instruction at the end of the subroutine performs an indirect jump: PC ← M[AR], which retrieves the saved return address and continues execution at the original calling point.
