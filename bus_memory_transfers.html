Common Bus System

Why Do We Need a Bus System?

A typical digital computer contains many registers. Transferring information between them directly via individual wires would be excessively complex and inefficient. The common bus system provides an elegant solution.

Key Concept: A bus structure consists of a set of common lines, one for each bit of a register, through which binary information is transferred one bit at a time.

Multiplexer Implementation

A common bus system can be constructed using multiplexers. For four 4-bit registers, we need four 4×1 multiplexers.

Selection Input Control:

  • S₁S₀ = 00: Register A is selected
  • S₁S₀ = 01: Register B is selected
  • S₁S₀ = 10: Register C is selected
  • S₁S₀ = 11: Register D is selected
BUS ← C // Content of register C is loaded onto the bus
R1 ← BUS // Content of the bus is loaded into register R1
R2 ← R1 // Combined operation: transfer from R1 to R2

Three-State Bus Buffers

Understanding Three-State Logic

An alternative to multiplexers, three-state gates provide an elegant solution for bus construction with better scalability.

Three Output States:
• Logic 0 (Low)
• Logic 1 (High)
• High-Impedance (Disconnected)

High-Impedance State

In the high-impedance state, the output is effectively disconnected and does not affect the circuit. It's like an open circuit, allowing multiple outputs to share a common bus line safely.

Buffer States:

Buffer 1: Disabled (Hi-Z)
Buffer 2: Disabled (Hi-Z)
Buffer 3: Disabled (Hi-Z)
Buffer 4: Disabled (Hi-Z)

Advantages of Three-State Buffers:

  • Multiple outputs can connect to the same bus line safely
  • Only one buffer is enabled at a time
  • Better scalability for large systems
  • Simplified control logic with decoders

Memory Transfers

Memory Unit Operations

The memory unit stores binary information in groups called words. Data transfer to and from memory is crucial for register transfer operations.

Memory Address Register (AR)

To access a specific word in memory, its address must be provided through the Memory Address Register (AR).



Memory Data Register (DR)

It holds the data to be written into memory or receives the data read from memory during a memory operation.

Memory Read Operation

Purpose: Transfer a word from memory to an external data register

Process:
1. Place the desired word address in AR
2. Memory unit provides content at that address to DR
DR ← M[AR] // Content of memory word M at address AR is transferred to Data Register DR

Memory Write Operation

Purpose: Transfer a word from a register into memory

Process:
1. Place the target address in AR
2. Place the data to be written in a data register (e.g., R1)
3. Transfer occurs from register to memory location
M[AR] ← R1 // Content of register R1 is transferred to memory word M at address AR
