⚡ Cache Coherence

Section 13.5 • Multiprocessor Systems

Understanding data consistency in shared memory architectures

📚 What is Cache Coherence?

In tightly coupled multiprocessor systems, each CPU has its own local cache memory for faster data access. While this dramatically improves performance, it creates a critical challenge: ensuring that all processors see the most recent value of shared data, regardless of which cache holds the copy.

💡 Key Insight: Cache coherence ensures that a read operation always returns the most recently written value to that memory address across all processors.

⚠️ Conditions for Incoherence

Cache incoherence occurs when cached copies of data become inconsistent. Here are the three primary conditions:

1Shared Writable Data

Two or more processors cache the same memory block containing modifiable data (not read-only).

2Write Operation

When one processor writes a new value, its cache updates but other processors' copies become stale.

3External I/O Write

An I/O device writes directly to main memory without notifying the caches, invalidating cached copies.

🔧 Solutions to Cache Coherence

🚌 Snooping Protocols (Bus Snooping)

Each cache controller monitors (or "snoops") all bus traffic. When a processor detects a write to a memory block it holds, it takes action:

✓ Write-Invalidate Protocol

Invalidates local copies in other caches when a write occurs. The next read will fetch the updated value from main memory or the writer's cache.

✓ Write-Update Protocol

Updates all cached copies with the new value immediately when a write occurs, keeping all caches synchronized.

📋 Directory-Based Protocols

Uses a central directory structure (typically in main memory) that tracks which caches hold copies of each memory block.

How it works:
  1. The directory maintains a record of which caches have copies of each block
  2. When a write occurs, the directory is consulted
  3. Targeted messages are sent only to caches with copies
  4. More efficient than broadcasting to all processors
