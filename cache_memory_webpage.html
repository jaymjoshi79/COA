⚡ Cache Memory

Fast, Small, and Essential: Understanding Computer Cache Architecture

What is Cache Memory?

Cache memory is a small, fast memory placed between the CPU and main memory. It exploits the principle of locality of reference, significantly reducing average memory access time and improving overall system performance.

🎯 Locality of Reference

Analysis of typical programs reveals two key patterns that make cache memory effective:

📍

Temporal Locality

Recently accessed memory locations are likely to be accessed again soon

📊

Spatial Locality

Memory locations near recently accessed locations are likely to be accessed

🔄

Loop Behavior

Program loops repeatedly access the same instructions and data

📈

90-10 Rule

Programs spend 90% of time executing only 10% of code

⚙️ How Cache Works

🔍 Basic Cache Operation ▼

When the CPU needs data:

  1. CPU sends address to check if word is in cache
  2. Cache Hit: Word found in cache → data sent to CPU immediately
  3. Cache Miss: Word not in cache → main memory accessed, entire block transferred to cache
  4. Block Transfer: Multiple words transferred together to exploit spatial locality

Average Access Time = Cache Time × Hit Rate + Main Memory Time × Miss Rate

Example: With 10ns cache, 1000ns main memory, and 90% hit rate:
Average = 10×0.9 + 1000×0.1 = 109ns (much better than 1000ns!)

📦 Block Size & Transfer ▼

Cache memory stores blocks of words rather than individual words. Block sizes vary from one word to about 16 words. When a miss occurs, the entire block containing the requested word is transferred from main memory to cache.

Why blocks? This exploits spatial locality - if you need one word, you'll likely need nearby words too!

🗺️ Cache Mapping Techniques

Three primary methods organize how main memory addresses map to cache locations:

1. Associative Mapping

Most Flexible

• Any block can go anywhere in cache
• Full address stored as tag
• All tags searched simultaneously
• Highest hit rate
• Most expensive hardware

2. Direct Mapping

Simplest & Fastest

• Each block has exactly one cache location
• Address = Tag + Index + Word
• Fast and simple
• Lower hit rate
• Conflicts if same index needed

3. Set-Associative

Best Compromise

• Cache divided into sets
• Block can go in any location within set
• Combines benefits of both methods
• 2-way, 4-way, 8-way common
• Balanced cost and performance

Address Structure Comparison

Mapping Type Address Components Search Method
Associative Tag (full) + Word Search all tags in parallel
Direct Tag + Index + Word Direct index lookup, compare one tag

✍️ Cache Write Operations

📝 Write-Through ▼

Updates both cache and main memory simultaneously

Advantages:

  • Main memory always has valid data
  • Simplifies I/O operations
  • No data loss if system fails

Disadvantage: Slower because every write waits for main memory

⚡ Write-Back ▼

Updates only cache initially, writes to memory only when block is replaced

Advantages:

  • Faster - no waiting for main memory
  • Multiple writes to same location only write once to memory
  • Used in typical programs (10-30% of references are writes)

Implementation: Uses "valid bit" to track if cache block has been modified

🔄 Replacement Algorithms

When cache is full and a new block is needed, which block should be replaced?

🎲

Random

Simplest method - randomly select a block to replace

🥇

FIFO

First In, First Out - replace oldest block in cache

⏰

LRU

Least Recently Used - most common, best performance

LRU Performance: Selects the item that has been unused for the longest time. Both FIFO and LRU algorithms provide similar hit rates, with LRU having a slight advantage as set size increases.

🚀 Cache Initialization

⚙️ Power-On Initialization ▼

When power is applied to the computer or when programs are loaded:

  1. Cache is empty and considered invalid
  2. Valid bits set to indicate no valid data
  3. First loads from main memory transfer blocks to cache
  4. Valid bits updated as blocks are loaded
  5. Subsequent accesses can hit cache

Important: The introduction of a new word automatically replaces invalid data. Thus initialization conditions are restored during normal operation.

📊 Performance Example

Real-World Scenario

System Configuration:

  • Cache access time: 10 nanoseconds
  • Main memory access time: 1000 nanoseconds
  • Hit ratio: 90% (typical for well-designed cache)

Performance Calculation:

Average access time = (0.9 × 10ns) + (0.1 × 1000ns) = 9ns + 100ns = 109ns

Result: Nearly 10× faster than accessing main memory directly! This is why cache is crucial for modern computer performance.

🎯 Key Takeaways

Speed Hierarchy

Cache sits between fast CPU and slower main memory, providing optimal balance

Locality Principle

Programs naturally exhibit temporal and spatial locality, making cache effective

Mapping Trade-offs

Associative = flexible, Direct = fast, Set-Associative = balanced compromise

Write Strategies

Write-through = safer, Write-back = faster for typical workloads
