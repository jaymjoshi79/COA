System Overview

A computer's operation can be understood through two primary representations that work together to provide a complete picture of its functionality:

🔄 Flowchart Representation

Visual sequence of operations showing the instruction cycle, interrupt handling, and control flow through different execution phases.

📝 Register Transfer Statements

Concise symbolic notation that describes all control functions and microoperations in a tabular format for precise implementation.

Instruction Cycle Flowchart

Complete Computer Operation Flow

1. Interrupt Check
Check R flip-flop: If R=1, enter interrupt cycle; If R=0, continue with instruction cycle
2. Interrupt Cycle (R=1)
RT0: AR ← 0, TR ← PC
RT1: M[AR] ← TR, PC ← 0
RT2: PC ← PC + 1, IEN ← 0, R ← 0, SC ← 0
3. Fetch & Decode (R=0)
T0: AR ← PC
T1: IR ← M[AR], PC ← PC + 1
T2: AR ← IR(0-11), I ← IR(15),
D0...D7 ← Decode IR(12-14)
4. Memory-Reference (D7=0)
Handle indirect addressing if I=1, then execute AND, ADD, LDA, STA, BUN, BSA, or ISZ instructions
5. Register-Reference (D7=1, I=0)
Execute CLA, CLE, CMA, CIR, INC, HLT instructions during D7I'T3, then SC ← 0
6. Input-Output (D7=1, I=1)
Execute INP, OUT, SKI, SKO, ION, IOF instructions during D7IT3, then SC ← 0

Register Transfer Statements

The complete computer operation can be precisely described using register transfer notation. This symbolic representation provides all necessary information for designing the computer's logic circuits.

Key Advantages

🎯 Precision

Exact specification of microoperations and control functions for unambiguous hardware implementation

🔧 Design Ready

Boolean functions derived directly from control statements enable logic circuit design

Sample Register Transfer Operations

Fetch Cycle:
T0: AR ← PC
T1: IR ← M[AR], PC ← PC + 1
T2: AR ← IR(0-11), I ← IR(15)
Interrupt Cycle:
RT0: AR ← 0, TR ← PC
RT1: M[AR] ← TR, PC ← 0
RT2: PC ← PC + 1, IEN ← 0, R ← 0, SC ← 0
Memory Reference (AND):
D0T4: DR ← M[AR]
D0T5: AC ← AC ∧ DR, SC ← 0
