Control Memory

Understanding Microprogrammed Control Organization

1Introduction to Control Memory

The Control Unit in a digital computer initiates a sequence of operations. A microprogrammed control unit uses microcode to generate control signals for executing instructions.

💡 Key Advantage: Unlike hardwired control units, microprogrammed units offer greater flexibility through software-updateable microcode!

Hardwired Control

Fixed by physical wiring
Requires hardware changes

VS

Microprogrammed Control

Software-updateable microcode
Greater flexibility

2Characteristics & Contents

🔒 Memory Type

📚 Content Organization

Contains microprograms (groups of microinstructions)

⚙️ Instruction Mapping

⚡Generating Microoperations

Microinstructions stored in control memory generate microoperations that perform fundamental tasks:

📥 Fetch Next Instruction
🎯 Evaluate Effective Address
⚙️ Execute Operation
🔄 Return to Fetch Phase
📝 Note: Each microinstruction contains bits that initiate microoperations and address bits that determine the next microinstruction address.

🔗Role in Address Sequencing

Control memory enables crucial address sequencing capabilities:

  • Incrementing: Control Address Register (CAR)
  • Branching: Conditional/unconditional based on status bits
  • Mapping: From instruction bits to control memory addresses
  • Subroutines: Call and return operations
