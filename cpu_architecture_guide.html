Central Processing Unit Introduction

The Central Processing Unit (CPU) is the brain of a computer system, responsible for executing instructions and performing calculations. It consists of several key components working together to process data and control the flow of information.

🧠 CPU Components

Key Functions:

  • Fetch instructions from memory
  • Decode instruction meaning
  • Execute operations
  • Store results

General Register Organization

Stack Organization

A stack is a Last-In-First-Out (LIFO) data structure used for managing function calls, local variables, and temporary data. It's essential for program execution and memory management.

Stack Operations:

  • PUSH: Add item to top of stack
  • POP: Remove item from top of stack
  • PEEK/TOP: View top item without removing
  • Stack Pointer (SP): Points to top of stack

Instruction Format

Instruction format defines how instructions are structured and encoded in machine language. Different formats optimize for various types of operations and addressing needs.

🔧 Instruction Format Examples

Three-Address Format

ADD
R1, R2, R3
Operation: R1 = R2 + R3

Two-Address Format

MOV
R1, R2
Operation: R1 = R2

One-Address Format

LOAD
100
Operation: AC = Memory[100]

Zero-Address Format (Stack-based)

ADD
Operation: Pop two values, add, push result

Format Components:

  • Opcode: Specifies the operation to perform
  • Operands: Specify data or addresses for the operation
  • Addressing Mode: How operands are interpreted

Addressing Modes

Addressing modes determine how the CPU interprets operand addresses in instructions. Different modes provide flexibility in accessing data and optimize for various programming patterns.

🎯 Addressing Mode Types

Direct Addressing

Address → Memory Location

Indirect Addressing

Address → Address → Data

Immediate Addressing

Operand = Data

Register Addressing

Register → Data

Indexed Addressing

Base + Index → Data

Relative Addressing

PC + Displacement → Address

RISC vs CISC

Two fundamental approaches to CPU instruction set design: Reduced Instruction Set Computer (RISC) and Complex Instruction Set Computer (CISC).

⚖️ Architecture Comparison

Aspect RISC CISC
Instruction Set Small, simple instructions Large, complex instructions
Instruction Length Fixed length (usually 32-bit) Variable length
Addressing Modes Few, simple modes Many, complex modes
Registers Many general-purpose registers Fewer, specialized registers
Execution Single cycle execution Multi-cycle execution
Pipeline Easy to pipeline Complex pipelining
Compiler Complex compiler design Simpler compiler design
Examples ARM, MIPS, PowerPC x86, x86-64

🚀 RISC Advantages

  • Higher performance
  • Lower power consumption
  • Simpler hardware design
  • Better for pipelining
  • More predictable timing

🔧 CISC Advantages

  • More compact code
  • Fewer instructions needed
  • Backward compatibility
  • Rich instruction set
  • Easier assembly programming
