CPU Instruction Format

Explore how instructions are structured and interact with different CPU organizations through interactive demonstrations and visualizations.

1CPU Organizations

The instruction format strongly relates to the CPU organization. Let's explore the three main types:

🏛️ General Register Organization

Utilizes multiple internal registers (R1, R2, R3, etc.). Instructions specify multiple addresses.

Example:
MOV R1, R2 : R1 ← R2
ADD R1, X : R1 ← R1 + M[X]

🎯 Single Accumulator Organization

Relies on a dedicated Accumulator register (AC). Instructions imply AC as operand/destination.

Example:
ADD X : AC ← AC + M[X]
Associated with One Address Instructions

📚 Stack Organization

Uses stack memory and Stack Pointer (SP). Operations implicitly push/pop values from stack.

Example:
PUSH
PUSH
ADD
POP
Associated with Zero Address Instructions

2Address Instruction Formats

Instructions are classified based on the maximum number of explicit address fields:

Instruction Type Structure Advantages Disadvantages
Three Address ADD R1, R2, R3 Very short programs Too many bits needed for addresses
Two Address MOV R1, R2 General format, flexible Moderate program length
One Address ADD X Simple, uses accumulator Longer programs needed
Zero Address ADD Very compact instructions Stack management overhead

4Instruction Comparison Summary

🎯 Zero Address

8 Instructions
Stack Organization
✅ Advantages:
  • Very compact instructions
  • No address field overhead
  • Simple instruction format
❌ Disadvantages:
  • Stack management overhead
  • More instructions needed
  • Complex expression evaluation

🎯 One Address

7 Instructions
AC Organization
✅ Advantages:
  • Simple with accumulator
  • One address per instruction
  • Easy to understand
❌ Disadvantages:
  • Accumulator bottleneck
  • Temporary storage needed
  • Sequential processing

🎯 Two Address

6 Instructions
Reg Organization
✅ Advantages:
  • Most common format
  • Good balance
  • Flexible addressing
❌ Disadvantages:
  • Two address fields
  • Moderate instruction length
  • Register management

🎯 Three Address

3 Instructions
Reg Organization
✅ Advantages:
  • Shortest programs
  • Maximum parallelism
  • Clear instruction meaning
❌ Disadvantages:
  • Long instruction format
  • Many address bits needed
  • Hardware complexity

5Key Takeaways

  • 🏗️ CPU Organization determines Instruction Format: The way a CPU is organized (registers, accumulator, or stack) directly influences how instructions are structured.
  • ⚖️ Trade-offs exist: Fewer addresses mean shorter programs but potentially more complex instruction sequences.
