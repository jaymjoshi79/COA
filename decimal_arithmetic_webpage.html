🔢 Why Decimal Arithmetic?

Computers naturally operate in binary, but humans think in decimal. There are two approaches:

Convert to Binary

Transform decimal → binary → compute → convert back

Direct Decimal

Perform arithmetic directly on coded decimal digits

The Binary-Coded Decimal (BCD) system allows computers to work directly with decimal digits by encoding each digit (0-9) using 4 bits.

🔄 9's and 10's Complement

For signed decimal arithmetic in BCD, we use complement systems similar to binary:

10's Complement Method

  1. Step 1: Find the 9's complement (subtract each digit from 9)
  2. Step 2: Add 1 to the least significant digit

Challenge: Computing 9's complement in BCD is difficult!

Solution: Use self-complementing codes (2421, Excess-3) where you simply flip all bits (1→0, 0→1)

📊 Key Advantages & Disadvantages

✓ Advantages

  • Direct decimal computation
  • No conversion overhead
  • Natural for human interface
  • Accurate decimal representation

✗ Disadvantages

  • More complex hardware
  • Difficult 9's complement
  • Less efficient storage
  • Slower operations
