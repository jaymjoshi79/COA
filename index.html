CPCentral Processing Unit

Understanding of CPU executes instructions using registers, stacks, instruction formats, addressing modes, and control operations, while comparing RISC and CISC architectures that shape modern processor design.

Cycle Phases:

  • General Register Organization: Multiple fast-access registers to hold data temporarily and speed up processing.
  • Stack Organization: Last-In-First-Out (LIFO) memory structure used for expression evaluation and subroutine management.
  • Addressing Modes: Locate operands for an instruction to provide flexibility in programming.

VPPipeline and Vector Processing

Modern CPUs speed up execution using parallel and pipelined instruction handling, RISC optimization, and vector/array data processing architectures to achieve high-performance computing.

Key Concepts:

  • Flynn’s Taxonomy: Classifies computer architectures based on instruction and data streams (SISD, SIMD, MISD, MIMD).
  • Parallel Processing: Multiple operations execute simultaneously to reduce overall computation time.
  • Pipelining: Breaks execution into stages so multiple instructions overlap in time for faster throughput.
  • Vector Processing: Same instruction operates on multiple data elements simultaneously (data-parallel).
  • Array Processors: Multiple processing units operate in parallel on array-structured data, often SIMD-based for large numerical tasks.

MOMemory Organization

Focuses on how computer systems manage the fundamental trade-off between speed, capacity, and cost. Since no single memory technology is simultaneously lightning-fast, infinitely large, and dirt cheap, the system uses a tiered approach to keep the CPU fed with data.

Design Elements:

  • Memory Hierarchy: The goal is to create a system that performs like fast, expensive memory but has the massive capacity of cheap storage.
  • Main Memory: The primary workspace (RAM) that communicates directly with the CPU.
  • Auxiliary Memory: Long-term, non-volatile storage such as hard drives and SSDs.
  • Associative Memory: A specialized type of memory where data is accessed by its content rather than a specific memory address.
  • Cache Memory: A small, ultra-fast buffer sitting between the CPU and Main Memory. It stores frequently accessed data to minimize the time the processor spends waiting for information from the slower RAM.
  • Virtual Memory: A management technique that treats secondary storage as an extension of RAM. It allows the system to run large applications that exceed the computer’s actual physical memory capacity.
