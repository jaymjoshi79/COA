⚡ Interprocessor Arbitration

Section 13.3 | Multiprocessor Systems

Interprocessor Arbitration manages access to shared resources in multiprocessor systems. When multiple processors compete for the system bus, arbitration logic ensures orderly, fair, and efficient access—preventing conflicts and maintaining system stability.
1

System Bus Architecture

▼

The System Bus is the shared communication highway connecting all processors and memory modules in a multiprocessor system. Since only one processor can use the bus at any given time, arbitration mechanisms are essential.

Serial Arbitration (Daisy Chain)

▼

In Serial Arbitration, bus grant signals propagate sequentially through a priority chain. The first requesting processor to receive the grant captures it, blocking further propagation.

How It Works:
Processors are connected in a priority chain (daisy chain). When the bus is requested, the grant signal travels down the chain. The first requesting processor blocks the signal, gaining bus control.

Parallel Arbitration Logic

▼

Parallel Arbitration uses a central arbiter with combinational logic to resolve conflicts instantly. All processors send requests simultaneously to the arbiter, which grants access to the highest-priority requester.

Architecture:
A central arbitration module receives all requests simultaneously. Using encoders, comparators, and decoders, it determines the highest-priority requester and issues a grant signal immediately—much faster than serial methods.

Dynamic Arbitration Algorithms

▼

Dynamic Arbitration goes beyond fixed priorities, adjusting priorities based on system state to ensure fairness and optimize performance.

Priority Adjustment Strategies:
  • Time-Based: Processors waiting longer receive higher priority
  • Round-Robin: Priority rotates cyclically to prevent starvation
  • Need-Based: Critical transactions get temporary priority boost
Algorithm Type Priority Basis Best Use Case
Time-Since-Access Waiting time Fair resource distribution
Round-Robin Cyclic rotation Equal processor importance
Need-Based Transaction type Real-time or I/O-heavy systems
Implementation Requirements:
  • Sophisticated control hardware
  • Counters to track access history
  • Logic to modify priorities dynamically
  • Prevents bus monopolization
