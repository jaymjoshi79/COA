🎯 What are Memory-Reference Instructions?

Memory-Reference Instructions are computer instructions that require an operand from memory to perform their operations. They are identified when the operation code (opcode) in bits 12-14 of the Instruction Register corresponds to codes D0 through D6.

7
Instruction Count

The basic computer supports exactly 7 memory-reference instructions, each with a unique opcode from D0 to D6.

⚡
Execution Phases

Instructions execute during timing signals T4, T5, and sometimes T6 after the addressing phase is complete.

🔄
Cycle Reset

After execution, the Sequence Counter (SC) is cleared to 0, returning to the fetch phase for the next instruction.

📍
Effective Address

Direct Addressing (I=0): The address part of the instruction (bits 0-11) is already in AR by T2.

Indirect Addressing (I=1): The effective address is fetched from memory at AR during T3.

⏱️
Execution Phase

Microoperations typically occur during timing signals T4 and T5 (and sometimes T6 for complex operations like ISZ).

🔄
Cycle Completion

Upon completion, SC ← 0 signals the control unit to return to the fetch phase (T0) for the next instruction.
