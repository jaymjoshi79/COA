Instruction Code

High-level machine instruction input

The starting point where machine instructions enter the control unit for processing

Mapping Logic

Converts instruction opcodes to addresses

Critical component that translates instruction opcodes into control memory addresses

Multiplexers

Route control signals based on conditions

Switch between different address sources based on branch logic and status bits

Control Address Register (CAR)

Holds current control memory address

Stores the address of the current microinstruction being executed

Control Memory

Stores microinstructions for execution

ROM or RAM containing microinstructions that control processor operations

Subroutine Register (SBR)

Saves return addresses for subroutines

Stack-like structure for managing microprogram subroutine calls

🔄 Original Instruction Mapping Diagram

This diagram illustrates the step-by-step mapping process from a machine instruction's opcode to the final microinstruction address in control memory.
🎯 Key Insight

The opcode bits (1011) are directly mapped to create the microinstruction address, with additional padding bits (0 and 00) to form the complete 7-bit address.

⚡ Flexibility

Don't care bits (x) in the mapping scheme dedicate a block of control memory addresses to an instruction's microprogram, allowing for future expansion of its microinstruction routine without altering the overall addressing structure.
