Multiprocessor Communication & Synchronization

Chapter 13.4 | Computer Architecture

Overview: In a multiprocessor system, multiple CPUs operate concurrently and often rely on shared resources such as memory for program execution. This introduces the fundamental requirement for mechanisms that allow processors to exchange data and coordinate their tasks to ensure correct program execution.

These mechanisms are categorized under Interprocessor Communication and Synchronization.

1
Interprocessor Synchronization
p. 507

Definition: Interprocessor Synchronization is necessary when multiple independent processors are executing tasks that must maintain a specific order or sequence relative to shared data.

Why it's necessary: Synchronization mechanisms are required to control the flow of execution among processes running on different CPUs, especially when resources are shared. For instance, if Processor A produces data that Processor B consumes, synchronization ensures that Processor B waits until the data is valid before attempting to read it.

🎯 Key Point

Synchronization techniques ensure that while processors run in parallel, access conflicts to shared memory (the core resource in a tightly coupled system) are managed effectively.

2
Mutual Exclusion with a Semaphore
p. 508

One of the most common applications of interprocessor synchronization is mutual exclusion, which ensures that only one processor accesses a specific shared resource (critical section) at a time.

Concept Explanation
Mutual Exclusion A requirement that prevents concurrent access to a shared data structure or hardware device by multiple processors simultaneously. Without it, race conditions occur where multiple processors update shared resources without coordination.
Semaphore A synchronization primitive used to manage access to a resource. It acts as a protected variable (integer counter or binary flag) that controls access to the critical section.
Atomic Operations Operations performed indivisibly in hardware to check and modify the semaphore's state, ensuring a processor cannot be interrupted while acquiring or releasing the lock.
