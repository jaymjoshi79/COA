I. Context and Characteristics

Page 489

The concept of multiprocessing introduces systems that utilize multiple processing units to increase computational power. This foundational section establishes the basic characteristics that define multiprocessor systems.

Key Focus: Understanding how multiple processors work together to enhance system performance and handle complex computational tasks efficiently.

II. Interconnection Structures

Section 13-2

Various interconnection structures are essential for linking multiple processors, enabling communication between processors and memory units.

Structure Type Page Description
Time-Shared Common Bus 491 Simplest method where all processors share a single communication path to memory
Multiport Memory 493 Memory units with multiple connection ports for simultaneous processor access
Crossbar Switch 494 Switching network providing dedicated paths between any processor and memory module
Multistage Switching Network 496 Complex, scalable networks for larger multiprocessor systems
Hypercube Interconnection 498 Specific topology for inter-processor connection in massively parallel processing
Trade-offs: From simple shared buses to complex switching networks, each structure balances cost, scalability, and performance differently.

III. Inter-processor Arbitration

Page 500

When processors share resources like a common bus, a mechanism is required to decide which processor controls the shared resource at any given time.

p. 500 System Bus

Defines the shared communication line and the basic context for arbitration.

p. 502 Serial Arbitration Procedure

Sequential method where processors request access one after another.

p. 503 Parallel Arbitration Logic

Faster method using combinational logic to resolve bus contention quickly.

p. 505 Dynamic Arbitration Algorithms

Algorithms allowing priority changes based on fairness, usage, or necessity.

Core Concepts: Priority Handling Fair Access Conflict Resolution

IV. Communication & Synchronization

Page 506

For multiple independent processors to execute tasks cooperatively, they must share data and coordinate operations effectively.

p. 507 Interprocessor Synchronization

Mechanisms to enforce correct sequencing of processes when sharing resources.

p. 508 Mutual Exclusion with a Semaphore

Software/hardware primitive ensuring only one processor accesses critical shared resources at a time.

Why It Matters: Without proper synchronization, multiple processors could corrupt shared data or create race conditions, leading to unpredictable system behavior.

V. Cache Coherence

Page 509

When multiple processors use local cache memory for speed, maintaining consistency across caches becomes a critical challenge.

p. 509 Conditions for Incoherence

Specific situations (e.g., sharing writable data) that lead to stale data in local caches.

p. 510 Solutions to Cache Coherence

Hardware or protocol methods to maintain consistency across all local caches.

The Problem: When multiple caches hold copies of the same memory block, updating one copy creates inconsistencies. This is the paramount challenge in tightly coupled multiprocessor systems.

VI. Shared Memory Multiprocessors

Implicit Focus

While not a distinct numbered section, the entire chapter focuses on this architectural type through its various topics.

Architectural Characteristics:
  • Interconnection structures focus on accessing common memory resources
  • Arbitration arises from contention for shared resources
  • Cache coherence is the paramount challenge in tightly coupled systems
  • Multiple CPUs share a central memory pool with local caches for speed
