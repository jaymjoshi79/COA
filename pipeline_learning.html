📋 What is Pipelining?

Pipelining is a computing technique that utilizes parallel processing principles to improve execution speed by transforming a sequential process into sub-operations.
⚡ Decomposition
Sequential process broken into smaller sub-operations
🎯 Dedicated Segments
Each sub-operation executed in special dedicated segment
🔄 Parallel Processing
Multiple computations progress simultaneously in distinct segments

🏗️ Pipeline Structure

Four Segment Pipeline

Segments (S) process data • Registers (R) synchronized by clock

Pipeline Characteristics:

  • Clock 1: T₁ starts in segment 1
  • Clock 2: T₁ moves to segment 2, T₂ starts in segment 1
  • Clock 3: T₁ moves to segment 3, T₂ to segment 2, T₃ starts in segment 1
  • Clock 4: T₁ completes (exits segment 4), pipeline continues filling
  • Clock 5 onwards: Pipeline is full - one task completes every clock cycle
  • Total time: 9 clock cycles to complete all 6 tasks
  • Throughput: 1 task per clock cycle (after pipeline fills at clock 4)

Input

⚡ Speed Up Calculation

S = n × tₙ / [(k + n - 1) × tₚ]

Where:

  • S = Speed Up factor
  • k = Number of segments in pipeline
  • n = Number of tasks
  • tₚ = Clock cycle time (pipeline)
  • tₙ = Time per task (non-pipeline)

🧮 Interactive Speed Up Calculator

📊 Key Insights:
  • When n ≫ k-1, Speed Up ≈ tₙ / tₚ
  • If tₙ = k × tₚ, then Maximum Speed Up = k; where k is the number of segment in pipeline
  • First task completes in k × tₚ time
  • After pipeline fills, one result per clock cycle
