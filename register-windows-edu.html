Overlapped Register Windows

Efficient Parameter Passing in RISC Processors

What Are Overlapped Register Windows?

Overlapped register windows are a characteristic feature of some RISC processors designed to optimize procedure calls by eliminating the time-consuming operations of saving and restoring register values. When a procedure is called, a new window of registers from the register file is allocated, with some registers overlapping between adjacent procedures for efficient parameter passing.

Key Benefit: Parameters and results can be passed between procedures without the actual movement of data, using only shared overlapping registers.

How It Works

Each procedure call results in the allocation of a new register window by incrementing a pointer. The return statement decrements the pointer and causes the activation of the previous window.

Global Registers

Shared by all procedures. Hold parameters common to all procedures.

Local Registers

Used for local variables within a procedure. Not shared with other procedures.

Common Registers

Overlapping registers shared between adjacent procedures for parameter passing.

Register Organization

In a typical implementation with 74 total registers:

  • 10 Global Registers (R0-R9): Available to all procedures
  • 64 Windowed Registers: Divided into 4 overlapping windows
  • 10 Local Registers per window: For local variable storage
  • 6 Common Registers: Shared between adjacent windows (6 low + 6 high overlap)

Example: Procedure A Calls Procedure B

When procedure A calls procedure B:

  • Registers R26-R31 are common to both A and B
  • Procedure A stores parameters in R26-R31
  • Procedure B uses local registers R32-R41 for its variables
  • To call procedure C, B passes parameters through R42-R47
  • Results are stored back in common registers for return
  • The register window pointer transfers control back to A
Circular Organization: Registers R10-R15 are common to procedures A and D because the four windows have a circular organization (being adjacent to D).

Mathematical Formulas

Window Size: L + 2C + G
Register File Size: (L + C)W + G

Where:

  • G = Number of global registers
  • L = Number of local registers in each window
  • C = Number of registers common to two windows
  • W = Number of windows

Advantages Over Traditional Methods

❌ Multiple Register Banks

Each procedure gets its own bank, but requires many physical register sets.

❌ Memory Stack

Stores parameters in memory, requiring memory access every time the stack is accessed.

✅ Overlapped Windows

Efficient parameter passing with minimal overhead and no data movement required.
