Register Transfer Language (RTL)

What is RTL?

Register Transfer Language (RTL) is a symbolic notation used to describe micro-operation sequences in a computer. It provides a concise and precise way to specify how hardware components interact.

Purpose: RTL enables designers to specify the physical organization of registers, the sequence of micro-operations, and the control that initiates these sequences.

Basic RTL Symbols:

  • Letters & Numerals: Denote registers (e.g., MAR, R2)
  • Parentheses ( ): Denote part of register (e.g., R2(0-7), R2(L))
  • Arrow ←: Denotes transfer of information
  • Comma ,: Separates micro-operations

Three-State Bus Buffers

A three-state gate has three output states:

  • Logic 0: Low output
  • Logic 1: High output
  • High-impedance: Disconnected state
When disabled: Output goes to high-impedance state
When enabled: Behaves as normal buffer (0 or 1)

Memory Transfers

Memory Read
DR ← M[AR]

Content of memory word M at address specified by AR is transferred to data register DR.

Memory Write
M[AR] ← DR

Content of DR is transferred to memory word M at address specified by AR.

Logic Micro-Operations

Bit-Manipulation Operations

Logic micro-operations perform bit-manipulation operations on data stored in registers, treating each bit as a binary variable.

Shift Micro-Operations

Serial Transfer of Data

Shift micro-operations are used for serial transfer of data and data-processing tasks by shifting register contents left or right.

Logical Shift
R1 ← shl R1 (Shift Left)
R2 ← shr R2 (Shift Right)

Transfers 0 into the serial input

Circular Shift (Rotate)
R ← cil R (Circular Left)
R ← cir R (Circular Right)

Preserves all bits by rotating them around

Arithmetic Shift
R ← ashl R (Arithmetic Left)
R ← ashr R (Arithmetic Right)

Shifts signed numbers (×2 left, ÷2 right)
