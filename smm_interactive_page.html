🖥️ Shared Memory Multiprocessors

Understanding Parallel Computing Architecture

Fundamental Concept

Definition: Shared Memory Multiprocessors (SMM) feature multiple independent CPUs that all have direct access to a common, shared memory space. This shared memory serves as the primary means of communication and data sharing among processors.

Classification of SMM

🔄 UMA (Uniform Memory Access)

  • All processors access memory in equal time
  • Symmetric architecture
  • Common bus or crossbar switch
  • Used in SMP systems
💡 Like a round table where everyone is equidistant from the center

⚡ NUMA (Non-Uniform Memory Access)

  • Memory access time varies by location
  • Local memory is faster than remote
  • Better scalability
  • Distributed memory banks
💡 Like offices connected by hallways - your own file cabinet is faster to access

Interconnection Networks

🔌

Common Bus

▼

Description: A single set of wires shared by all components. Only one processor can use the bus at a time.

Application: Primarily UMA (SMP)

Scalability: Low due to bus contention

🔀

Crossbar Switch

▼

Description: High-speed switches allowing any processor to connect to any memory module simultaneously.

Application: UMA systems

Scalability: Medium/High - Excellent performance but O(N²) complexity

🌐

Multistage Interconnection Network (MIN)

▼

Description: Network of switches in multiple stages providing multiple paths between processors and memory.

Examples: Omega Network, Banyan Network

Application: UMA or NUMA

Scalability: High - Best balance of cost, complexity, and performance

Cache Coherence Problem

⚠️ The Problem

Each processor has its own private cache. When one processor modifies shared data, other processors may hold stale copies, leading to an inconsistent view of memory.

Example Scenario:
  1. P₁ reads X=5 → X loaded into P₁'s cache
  2. P₂ reads X=5 → X loaded into P₂'s cache
  3. P₁ executes X ← X+1, so X=6 in P₁'s cache
  4. ❌ P₂'s cache still shows X=5 (Incoherence!)

✅ Solutions

1. Snooping Protocols (Bus-based - UMA)

  • All cache controllers monitor the shared bus
  • Write operations are broadcast on the bus
  • Invalidate Protocol: Invalidate other caches' copies (most common)
  • Write-Update Protocol: Update other caches with new data

2. Directory-Based Protocols (Large Scale - NUMA)

  • Maintains a distributed directory tracking cache copies
  • Sends point-to-point messages only to relevant caches
  • More scalable with reduced interconnect traffic
  • Acts as centralized agent for invalidation/updates

Advantages & Disadvantages

Aspect ✓ Advantages ✗ Disadvantages
Programming Easy to program with explicit/implicit memory sharing UMA systems suffer from bus contention and CPU limits
Communication Low-latency communication via memory access Cache coherence adds complexity and overhead
Data Sharing Efficient access to large shared data structures Synchronization primitives required (locks, semaphores)
Performance Fast data exchange between processors Potential for deadlocks and synchronization overhead
