🔄 Parameter Passing Methods

Call by Reference

The subroutine accesses parameters through memory addresses. Changes made in the subroutine affect the original data.

BSA (Branch and Save Address)

Stores the return address in the first location of the subroutine and jumps to the next location to begin execution.

Indirect Addressing

Uses the 'I' suffix to access data indirectly through pointers, enabling flexible parameter passing.

Execution Steps:

1 LDA X: Load first operand (7B95) into AC
2 BSA OR: Save return address (202) in OR, jump to 208
3 CMA: Complement AC (7B95 → 846A)
4 STA TMP: Store complemented first operand in TMP
5 LDA OR I: Load second operand (3AF6) using indirect addressing
6 CMA: Complement second operand (3AF6 → C509)
7 AND TMP: AND the two complemented operands
8 CMA: Final complement to get OR result
9 Return: ISZ OR and BUN OR I to return with result


🧮 De Morgan's Law Implementation

This subroutine implements the OR operation using De Morgan's Law:

A OR B = NOT(NOT A AND NOT B)

The algorithm:

  1. Complement operand A
  2. Complement operand B
  3. AND the two complements
  4. Complement the result

📊 Key Advantages

  • Reusability: Subroutine can be called multiple times
  • Modularity: Separates functionality into distinct units
  • Parameter Passing: Flexible data exchange mechanism
  • Memory Efficiency: Single copy serves multiple calls
  • Maintainability: Changes in one place affect all calls
