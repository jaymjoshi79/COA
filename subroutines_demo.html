💡 Key Concepts

🎯 What is a Subroutine?

A subroutine is a reusable block of code that performs a specific task. Instead of repeating code, we can call the subroutine multiple times.

🔄 BSA Instruction

Branch and Save Address - Saves the return address and jumps to the subroutine. Essential for knowing where to return!

↩️ Return Mechanism

BUN SH4 I - Uses indirect addressing to return to the saved location in the main program.

⬅️ Left Shift Operation

The subroutine shifts bits left by 4 positions using CIL (Circulate Left) instructions, effectively multiplying by 16.

🔍 Deep Dive: How It Works

📖 Program Flow

This program demonstrates the power of subroutines by using the same shift-left-4 routine twice:

  1. First, it shifts the value in X (1234₁₆) left by 4 bits
  2. Then, it shifts the value in Y (4321₁₆) left by 4 bits
  3. The subroutine SH4 handles both operations, avoiding code duplication

⚙️ Technical Details

CIL (Circulate Left): Rotates all bits in the accumulator one position left. The leftmost bit moves to the rightmost position.

AND MSK: Uses mask FFF0₁₆ to clear the rightmost 4 bits, ensuring a clean left shift (not rotation).

Mathematical Effect: Left shifting by 4 bits is equivalent to multiplying by 2⁴ = 16.
