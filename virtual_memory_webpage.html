What is Virtual Memory?

Virtual memory is a crucial memory management technique that provides an "idealized abstraction" of storage resources, separating logical memory from physical memory.

📚 Key Concept

Virtual memory allows programs to use more memory than physically available by utilizing specialized hardware to manage memory resources efficiently through a memory hierarchy.

Why Virtual Memory?

🔄 Program Isolation

Each program operates in its own virtual address space, preventing interference

📈 Memory Expansion

Run programs larger than physical memory by swapping pages

🛡️ Protection

Hardware-enforced memory protection between processes

🎯 Simplified Programming

Programmers work with consistent virtual addresses

What are Pages?

Address Translation Process

Step 1: CPU generates virtual address
Step 2: Extract page number from virtual address
Step 3: Search TLB for page number (parallel search)
Step 4: If TLB hit → use frame number; If TLB miss → access page table
Step 5: Combine frame number with page offset
Step 6: Access physical memory at computed address

Page Replacement Algorithms

Page Fault

When a program references a page not currently in physical memory, a page fault occurs. The operating system must load the required page from auxiliary storage.

Common Replacement Algorithms

🔄 FIFO (First-In-First-Out)

Replace the oldest page in memory. Simple but may remove frequently-used pages.

🎯 LRU (Least Recently Used)

Replace the page that hasn't been used for the longest time. Better performance but requires tracking.

🎲 Optimal

Replace the page that won't be used for the longest time in the future. Theoretical best, impractical in reality.

⚡ Clock (Second Chance)

Circular list with reference bits. Approximates LRU with less overhead.
